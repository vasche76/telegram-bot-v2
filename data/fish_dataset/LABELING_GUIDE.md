# Руководство по сборке датасета рыб

Это пошаговая инструкция для сбора фотографий и подготовки данных для обучения модели распознавания рыб. Никаких технических знаний не требуется — просто следуйте шагам.

---

## Шаг 1 — Сбор фотографий

**Что фотографировать:**

- **Целую рыбу (whole_fish):** рыба полностью видна на снимке — с головы до хвоста. Можно в руках, на земле, на лодке, в воде.
- **Приманку (lure):** блесна, воблер, мушка или любая другая приманка — без рыбы на фото.
- **Часть рыбы (fish_part):** виден только хвост, голова или бок рыбы, но не вся рыба целиком.
- **Малька (fry):** очень маленькая рыбка, мальки.
- **Нет рыбы (no_fish):** вода, берег, трава, лодка — любой снимок без рыбы и приманки.

**Советы по съёмке:**

- Делайте фото с разных углов: сверху, сбоку, немного снизу.
- Снимайте при разном освещении: солнечно, пасмурно, в тени.
- Фотографируйте рыбу мокрой и сухой, в руках и без.
- Чем больше разнообразия — тем лучше работает модель.
- Минимум на каждый класс: **20 фото** (лучше — 50 и больше).

**Куда складывать фото (Stage A — для определения что на снимке):**

```
data/fish_dataset/stage_a/raw/whole_fish/   ← целая рыба
data/fish_dataset/stage_a/raw/lure/         ← приманка
data/fish_dataset/stage_a/raw/fish_part/    ← часть рыбы
data/fish_dataset/stage_a/raw/fry/          ← малёк
data/fish_dataset/stage_a/raw/no_fish/      ← нет рыбы
```

**Куда складывать фото (Stage B — для определения вида рыбы):**

```
data/fish_dataset/stage_b/pike/             ← щука
data/fish_dataset/stage_b/taimen/           ← таймень
data/fish_dataset/stage_b/grayling/         ← хариус
data/fish_dataset/stage_b/whitefish/        ← сиг
data/fish_dataset/stage_b/perch/            ← окунь
data/fish_dataset/stage_b/unknown_fish/     ← рыба неизвестного вида
```

Для Stage B разметка не нужна — просто положите фото в нужную папку с названием вида. Минимум: **15 фото** на вид (лучше — 30 и больше).

---

## Шаг 2 — Регистрация на Roboflow

Roboflow нужен только для Stage A (чтобы нарисовать рамки вокруг объектов на фото).

1. Откройте браузер и перейдите на сайт: **https://roboflow.com**
2. Нажмите кнопку **"Sign Up"** (Зарегистрироваться).
3. Создайте аккаунт — бесплатно, можно через Google.
4. После входа нажмите **"Create New Project"** (Создать новый проект).
5. Введите название, например: `fish-detection-stage-a`.
6. Тип проекта выберите: **"Object Detection"** (Обнаружение объектов).
7. Нажмите **"Create Project"**.

---

## Шаг 3 — Загрузка фото и разметка рамок (Stage A)

**Загрузка фотографий:**

1. В проекте нажмите **"Upload"** (Загрузить).
2. Перетащите фотографии из папок `stage_a/raw/` или выберите их через диалог.
3. Дождитесь загрузки и нажмите **"Save and Continue"**.

**Как рисовать рамки (bounding boxes):**

1. Откройте фото в редакторе Roboflow (нажмите на снимок).
2. Выберите инструмент рамки (квадратик в левой панели) или нажмите клавишу **B**.
3. Нарисуйте прямоугольник вокруг объекта — зажмите мышь и тяните.
4. Появится меню выбора класса. Введите или выберите нужный класс:
   - `whole_fish` — целая рыба
   - `lure` — приманка
   - `fish_part` — часть рыбы
   - `fry` — малёк
   - `no_fish` — нет рыбы (если нужно явно пометить пустой снимок)
5. Нажмите **Enter** или кликните на класс.
6. Если на фото несколько объектов — нарисуйте рамку вокруг каждого.
7. Перейдите к следующему фото стрелкой вправо или клавишей **D**.

**Советы по рамкам:**

- Рамка должна плотно облегать объект — не слишком большая и не обрезанная.
- Если рыба частично закрыта — всё равно рисуйте рамку вокруг видимой части.
- Не нужно обводить каждый плавник — достаточно общего контура рыбы.

---

## Шаг 4 — Экспорт в формате YOLO

После разметки всех фото нужно выгрузить данные в формате, который понимает модель.

1. В меню слева выберите раздел **"Versions"** (или кнопку **"Generate"**).
2. Настройте разбивку датасета:
   - Train: **70%**
   - Valid: **20%**
   - Test: **10%**
3. Нажмите **"Generate"** и дождитесь обработки.
4. После генерации нажмите **"Export Dataset"**.
5. В списке форматов выберите: **"YOLO v8"** (или "YOLOv5 PyTorch" — оба подойдут).
6. Убедитесь, что выбран вариант **"Download zip to computer"**.
7. Нажмите **"Continue"** и сохраните архив `.zip` на компьютер.

---

## Шаг 5 — Куда поместить экспортированные файлы

1. Распакуйте скачанный архив `.zip`.
2. Внутри будут папки `train/`, `valid/` (или `val/`), `test/`.
3. Скопируйте содержимое в следующие папки проекта:

```
Из архива train/images/  →  data/fish_dataset/stage_a/labeled/train/images/
Из архива train/labels/  →  data/fish_dataset/stage_a/labeled/train/labels/
Из архива valid/images/  →  data/fish_dataset/stage_a/labeled/val/images/
Из архива valid/labels/  →  data/fish_dataset/stage_a/labeled/val/labels/
Из архива test/images/   →  data/fish_dataset/stage_a/labeled/test/images/
Из архива test/labels/   →  data/fish_dataset/stage_a/labeled/test/labels/
```

Важно: копируйте именно содержимое папок (фото и `.txt` файлы), а не сами папки целиком.

---

## Шаг 6 — Stage B: просто раскладываем фото по папкам

Для определения вида рыбы никакой разметки не нужно.

1. Сфотографируйте рыбу (желательно целую, хорошо видно вид).
2. Положите фото в соответствующую папку:

```
data/fish_dataset/stage_b/pike/          ← щука
data/fish_dataset/stage_b/taimen/        ← таймень
data/fish_dataset/stage_b/grayling/      ← хариус
data/fish_dataset/stage_b/whitefish/     ← сиг
data/fish_dataset/stage_b/perch/         ← окунь
data/fish_dataset/stage_b/unknown_fish/  ← не знаете вид — сюда
```

Название папки = название вида. Больше ничего делать не нужно.

**Рекомендации:**

- На каждый вид — минимум 15 фото, лучше 30 и больше.
- Разные ракурсы, разное освещение, разные особи.
- Чем больше разнообразия — тем точнее определение.

---

## Шаг 7 — Проверка, что всё готово

После того как добавили фото, запустите скрипт проверки. Он покажет сколько фото в каждом классе и скажет, всё ли готово к обучению.

**Как запустить (в терминале из папки проекта):**

```bash
python3 scripts/validate_dataset.py
```

**Что означает вывод:**

- `OK` — всё хорошо, достаточно фото.
- `WARN` — фото есть, но маловато. Рекомендуется добавить ещё.
- `ERROR` — критически мало фото, нельзя обучать.
- `Result: PASS` — датасет готов к обучению.
- `Result: FAIL` — нужно исправить ошибки перед обучением.

**Дополнительные скрипты:**

```bash
# Подробная статистика и диаграмма
python3 scripts/dataset_stats.py

# Поиск дублирующихся фото
python3 scripts/check_duplicates.py

# Удалить дубликаты автоматически
python3 scripts/check_duplicates.py --delete
```

## Частые вопросы

**Можно ли использовать фото из интернета?**
Лучше использовать собственные фотографии с реальных рыбалок — модель будет точнее работать именно с вашими снимками. Фото из интернета можно добавить для увеличения разнообразия, но они не должны быть единственным источником.

**Какое качество фото нужно?**
Достаточно обычного смартфона. Фото не обязательно должны быть идеальными — нечёткие, тёмные или немного смазанные снимки тоже полезны, они помогают модели работать в реальных условиях.

**Что делать если рыба не попадает ни в один вид?**
Кладите фото в папку `unknown_fish`. Для таких фото минимумов нет.

**Нужно ли одну и ту же рыбу фотографировать несколько раз?**
Да — с разных углов и при разном освещении. Но лучше иметь фото разных особей, а не только одной рыбы во многих позах.
